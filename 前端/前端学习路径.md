# 前端学习路径（从小白到资深实战专家）

## 一、面向实战的量化成果（SLO/SLA）
- 性能：`LCP P75 ≤ 2.5s`、`FID P75 ≤ 100ms`、`CLS P75 ≤ 0.1`；路由切换 `P95 ≤ 200ms`，首屏 JS `≤ 180KB gz`。
- 可用性与稳定性：前端错误率 `<0.1%`，崩溃会话 `≥ 99.9%`；接口失败降级可用。
- 可访问性：Lighthouse Accessibility `≥ 90`，axe `0` 条 Critical 违规，键盘导航覆盖核心交互。
- SEO：Lighthouse SEO `≥ 90`，关键页面可被抓取与渲染，元数据合规。
- 测试：单测/集成覆盖率 `≥ 70%`（语句），关键 E2E 冒烟 `100%` 通过。
- 安全：无反射型 XSS，CSP 启用；依赖库 Critical 漏洞 `0`；Cookie `HttpOnly/SameSite/Secure` 合规。
- 交付：CDN 命中率 `≥ 95%`，构建产物稳定回滚；发布无停机（蓝绿/灰度）。

## 二、里程碑项目（逐层爬坡）
1) 高性能电商 SPA（SSR/CSR 混合）：商品/购物车/结算闭环；流式 SSR、骨架屏与懒加载；Playwright 覆盖核心交易流。
2) 设计系统与组件库：基于 Storybook 的可访问组件（ARIA 合规）、主题/设计令牌、树摇优化；内网 npm 发布与版本治理。
3) 微前端门户：Module Federation/Turborepo，两个子应用独立部署与单点登录；共享设计系统与监控。
（可选加分）实时协同编辑器：WebSocket/WebRTC + CRDT（如 Yjs），冲突解决与离线同步。

## 三、核心技能图谱
- HTML/CSS：语义化、Flex/Grid、响应式/移动端适配、BEM/Tailwind/SCSS、动画与过渡。
- JavaScript/TypeScript：类型系统、模块化、事件循环/异步、Promise/RxJS、错误边界与重试退避。
- 浏览器与网络：DOM/事件、Storage/Cache API、Service Worker、HTTP/CORS/缓存/ETag、WebSocket。
- 框架与渲染：React/Vue 基础与高级（Hooks/Composition）、路由/表单、SSR/SSG/ISR、Hydration。
- 状态管理：Redux Toolkit、Zustand/Recoil/Pinia；服务端状态（TanStack Query），缓存失效与并发治理。
- 性能优化：代码分割、预加载/预连接、图片优化（响应式图像）、关键路径与性能预算。
- 测试与质量：Jest/Vitest、Testing Library、Playwright/Cypress、Lighthouse CI、axe 可访问性检测。
- 构建与工程化：Vite/Webpack、Babel/TS、ESLint/Stylelint/Prettier、Husky/Commitlint、约定式提交。
- 安全与合规：XSS/CSRF、CSP/SRI、SameSite/Secure、依赖安全扫描（npm audit/Snyk）。
- PWA：离线缓存/背景同步、安装性与更新策略、推送与通知。
- 观测与治理：Web-Vitals、RUM、Sentry/前端日志、特性开关与灰度发布。
- 微前端与单仓：Module Federation、Monorepo（pnpm workspace、Turborepo/Nx）、版本与依赖治理。
- CI/CD 与交付：GitHub Actions、构建与产物管理、CDN/边缘函数部署、预览环境与回滚。

## 四、环境与工具栈
- Node `≥ 20`、`pnpm`/npm，Vite 5、TypeScript 5.6、ESLint/Prettier/Stylelint、Husky/Commitlint、Storybook、Jest/Vitest、Playwright、Lighthouse/axe。
- DevTools：Chrome DevTools、WebPageTest、Bundle Analyzer、Source Map 可观测；Postman/Insomnia 接口调试。

## 五、交付与验收模板
- 仓库结构：`apps/`（业务）、`packages/design-system/`（组件库）、`packages/utils/`、`scripts/`、`docs/`。
- 每次迭代提交：代码、指标截图（Lighthouse/Web-Vitals）、测试报告、验收结论；风险与改进项。
- 项目文档：`README`（运行/构建/部署）、`ARCHITECTURE.md`（关键决策与权衡）、`OPERATIONS.md`（监控与告警）。
